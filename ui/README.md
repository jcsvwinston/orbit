# admin/ui

React + Connect-Web UI for the Nucleus admin observability server. The
production build is embedded into the admin server binary via
`//go:embed all:dist` (see `admin/server/ui/embed.go`); the dev server
runs separately on :5173 with a Vite proxy.

## Day-to-day commands

```bash
# Hot-reload dev server. Proxies /nucleus.admin.v1.* and /healthz to
# the admin server on :8080 and injects X-Auth-User: dev so the UI
# listener (which trusts 127.0.0.1) accepts the request without a real
# reverse proxy.
npm run dev

# Production build. Writes directly to ../server/ui/dist so the next
# `make server-build` picks it up via go:embed.
npm run build

# Strict TypeScript type-check (no emit).
npm run typecheck

# ESLint with the strict bans (no any, no unsafe-*).
npm run lint
```

`make ui-dev`, `make ui-build`, `make ui-typecheck`, `make ui-lint`
from the repo root are thin wrappers that cd into this directory.

## Pages

* `/#/dashboard` — connected node count, version mix.
* `/#/nodes` — full registry table with labels and status.
* `/#/http` — live HTTP request stream.
* `/#/sql` — live SQL statement stream.
* `/#/sessions` — created/touched/destroyed session lifecycle.

Hash-based routing (no `react-router-dom`); the small surface area
isn't worth the dep.

## TypeScript hygiene

`tsconfig.app.json` enables `strict: true`,
`exactOptionalPropertyTypes: true`, `noUncheckedSideEffectImports: true`
and `noImplicitOverride: true`. The ESLint config errors on
`@typescript-eslint/no-explicit-any` and the
`@typescript-eslint/no-unsafe-*` family. We do not bypass these rules
in this UI; if you hit one, the right fix is almost always a richer
type, not an escape hatch.

## Bundle size

Current production bundle (Vite 6, prod mode, no source maps):

| File                                   | Raw    | Gzip    |
|----------------------------------------|-------:|--------:|
| `index.html`                           | 0.5 KB | 0.3 KB  |
| `assets/index-*.css`                   | 12 KB  | 3.1 KB  |
| `assets/index-*.js`                    | 325 KB | 96.7 KB |

Roughly half of the JS bundle is React 19 + TanStack Query;
Connect-Web + protobuf adds another ~70 KB. Acceptable for an internal
operator tool; if a future need arises we can split out
`@tanstack/react-query` and `recharts` into a vendor chunk via Vite
`build.rollupOptions.output.manualChunks`.
